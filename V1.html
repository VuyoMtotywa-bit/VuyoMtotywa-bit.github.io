Employee Payroll System
What it is

A payroll system demonstrating advanced OOP through an employee class hierarchy. An abstract base class defines what all employees share, while two specialised child classes handle different employment types — each calculating net salary differently. The same method name produces different results depending on which type of employee calls it — this is polymorphism.

  • Abstract base class ClsEmployee marked MustInherit — cannot be used directly, only inherited
  • ClsFullTimeEmployee calculates monthly pay as Annual Salary ÷ 12
  • ClsCasualEmployee calculates pay as Pay Rate × Hours Worked
  • Both override CalcNetSalary() — same method, different logic = polymorphism
  • Option Strict On enforced — professional type-safe coding standard
